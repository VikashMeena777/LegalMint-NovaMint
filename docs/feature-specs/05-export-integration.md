# Feature Specification: Export & Integration

**Feature ID:** FEAT-005
**Version:** 1.0
**Status:** Draft
**Last Updated:** 2026-05-13

---

## 1. Feature Overview and Purpose

The Export & Integration module enables users to take their LegalMint AI-generated documents and compliance data outside the platform. It provides professional-grade PDF and DOCX exports of legal documents, and a lightweight embeddable HTML snippet for publishing policies directly on users' websites. All exports maintain legal document formatting standards and are tier-gated based on subscription level.

### Key Objectives
- Generate print-ready, professionally formatted PDFs of all legal documents
- Generate editable DOCX files for further customization in word processors
- Provide an embeddable HTML/JS snippet for privacy policies and terms that auto-updates
- Apply consistent legal document styling (letterhead, table of contents, proper typography)
- Enforce subscription tier access controls on export formats
- Handle edge cases like long documents, special characters, and embedded links

## 2. PDF Generation Pipeline

### Architecture

```
┌──────────────────────────────────────────────────────────────────────┐
│                        PDF GENERATION PIPELINE                        │
└──────────────────────────────────────────────────────────────────────┘

  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
  │  DOCUMENT   │    │  PDF TEMPLATE│    │   RENDER    │    │  POST-      │
  │  MARKDOWN   │───▶│  + STYLING  │───▶│   ENGINE    │───▶│  PROCESSING │
  │  (source)   │    │  (HTML/CSS) │    │  (Puppeteer)│    │             │
  └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                                  │
                                                                  ▼
                                                          ┌─────────────┐
                                                          │  PDF FILE   │
                                                          │  (.pdf)     │
                                                          └─────────────┘
```

### Step-by-Step Pipeline

#### Step 1: Markdown to HTML Conversion
- Source document (stored as Markdown) is converted to HTML using a Markdown renderer (e.g., `marked` or `markdown-it` with legal extensions)
- Extensions applied:
  - Auto-numbering of headings (1, 1.1, 1.1.1) for legal section numbering
  - Smart quotes, en/em dashes, and other typographic niceties
  - Footnote support (rendered as page footnotes in PDF)
  - Table of Contents generation from heading anchors

#### Step 2: PDF Template Application
- The HTML is wrapped in a print-optimized HTML template with CSS styling:
  - Page size: US Letter (8.5" × 11") or A4 (210mm × 297mm) — user-selectable
  - Margins: 1" top/bottom, 1" left/right (legal document standard)
  - Typography: Serif font (Georgia or Times New Roman), 11pt body, appropriate line height
  - Running header: Document title + page number on every page
  - Letterhead: Company name and logo at top of first page
  - Page breaks before each major section
  - Avoid widow/orphan lines (CSS `widows: 3; orphans: 3;`)

#### Step 3: Puppeteer Rendering
- The styled HTML is passed to a headless Chromium instance via Puppeteer
- Rendering configuration:
  ```javascript
  await page.pdf({
    path: outputPath,
    format: 'Letter',         // or 'A4'
    margin: {
      top: '1in',
      bottom: '1in',
      left: '1in',
      right: '1in'
    },
    printBackground: true,    // preserve background colors
    displayHeaderFooter: true,
    headerTemplate: `
      <div style="font-size: 9px; text-align: center; width: 100%;">
        <span class="title"></span> — Page <span class="pageNumber"></span>
      </div>`,
    footerTemplate: `
      <div style="font-size: 8px; text-align: center; width: 100%; color: #888;">
        Generated by LegalMint AI | <span class="date"></span>
      </div>`,
    preferCSSPageSize: true
  });
  ```

#### Step 4: Post-Processing
- **Table of Contents:** Generate a clickable (internal links) and numbered TOC on page 2
- **Metadata injection:** Set PDF metadata (Title, Author = Company Name, Subject = Document Type, Keywords)
- **Page numbering:** Ensure continuous numbering; TOC pages use Roman numerals (i, ii, iii), body uses Arabic (1, 2, 3)
- **Accessibility tagging:** Tag PDF for screen readers (heading levels, reading order, alt text)
- **File size optimization:** Compress embedded images; target < 5MB for standard documents

### Technology Choice: Puppeteer
Puppeteer is preferred over React-pdf for this use case because:
- Full CSS print styling support (React-pdf has limited CSS support)
- Reliable page break handling
- Better typography rendering
- Header/footer templates for running heads
- Ability to match browser-compatible styling (same engine as the embeddable snippet)

## 3. DOCX Generation Pipeline

### Architecture

```
  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
  │  DOCUMENT   │    │  DOCX       │    │  DOCX       │    │  FINAL      │
  │  MARKDOWN   │───▶│  STRUCTURE  │───▶│  STYLING    │───▶│  DOCX FILE  │
  │  (source)   │    │  BUILDER    │    │  (OXML)     │    │  (.docx)    │
  └─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
```

### Step-by-Step Pipeline

#### Step 1: Parse Markdown into Document AST
- Parse the document Markdown into an Abstract Syntax Tree (AST) using a Markdown parser
- The AST preserves: headings (levels), paragraphs, lists (ordered/unordered/nested), bold/italic formatting, links, tables, blockquotes (for legal references)

#### Step 2: Build DOCX Structure
- Using a library like `docx` (Node.js) or `python-docx`, map the AST to OOXML elements:
  - `## Heading` → Word paragraph with "Heading 2" style
  - Bullet list → Word bullet list with appropriate indentation
  - Bold text → Run with bold property
  - Links → Hyperlink element (clickable in Word)
  - Tables → Native Word tables with styled headers

#### Step 3: Apply DOCX Styling
- **Document-level styles:**
  - Default paragraph font: Times New Roman or Calibri, 11pt
  - Heading styles: built-in Word heading styles (Heading 1, Heading 2, etc.) for automatic TOC support
  - Section breaks between major document sections
  - Letterhead: Company name + logo in Word header (first page only)
  - Footer: Page number (centered, "Page X of Y" format), "Generated by LegalMint AI" on first page footer
  - Line spacing: 1.15 (Word default) or 1.5 (legal preference)
- **TOC:** Insert a Word Table of Contents field (`<w:toc/>`) that renders as an automatic TOC when the user opens the file in Word
- **Cover page:** Optional — first page with document title, company name, date, and disclaimer

#### Step 4: Generate and Stream
- The DOCX builder serializes to a `.docx` file (ZIP of OOXML)
- File is streamed to the user's browser as a download via `Content-Disposition: attachment; filename="Privacy Policy - YourCompany.docx"`

### Key DOCX Features
| Feature | Implementation |
|---------|---------------|
| Automatic TOC | `<w:toc/>` OOXML field; user clicks "Update Table" in Word |
| Page numbering | OOXML footer with `PAGE` / `NUMPAGES` fields |
| Clickable hyperlinks | OOXML hyperlink relationships |
| Track Changes ready | Document is not protected; user can enable Track Changes in Word |
| Editable content | All content is editable; no locked/restricted sections |
| Embedded images | Company logo embedded as inline image in header |

## 4. Embeddable HTML Snippet Design

### Purpose
The embeddable snippet allows users to publish their Privacy Policy, Terms of Service, or Cookie Policy directly on their own website without manual copy-paste. When the user updates the document in LegalMint AI, the embedded version updates automatically.

### Snippet Code (User-Facing)
```html
<!-- LegalMint AI Privacy Policy Embed -->
<div id="legalmint-policy" 
     data-document-id="doc_abc123" 
     data-theme="light" 
     data-language="en">
</div>
<script async 
        src="https://cdn.legalmint.ai/embed/v1/legalmint-embed.js"
        data-client-id="client_xyz789">
</script>
```

### Design Principles
1. **Lightweight:** The embed script should be < 30KB gzipped
2. **Self-contained:** No external CSS dependencies; all styles are inline/scoped
3. **Shadow DOM:** Renders inside a Shadow DOM to prevent style conflicts with the host page
4. **Automatic Updates:** Polls the LegalMint API periodically (every 6 hours) for document updates
5. **Graceful Fallback:** If the script fails to load or the API is unreachable, the embedded div is empty and invisible (no broken UI on the host page)
6. **Caching:** Uses browser Cache API or localStorage to serve cached version during API outages

### Snippet Configuration Options

| Data Attribute | Type | Default | Description |
|---------------|------|---------|-------------|
| `data-document-id` | String | Required | ID of the document to embed |
| `data-theme` | String | `"light"` | `"light"` or `"dark"` — matches host page theme |
| `data-language` | String | `"en"` | Language code (en, es, fr, de, pt) |
| `data-show-disclaimer` | Boolean | `true` | Show/hide the AI generation disclaimer |
| `data-show-header` | Boolean | `true` | Show/hide document title and last-updated date |

### Rendered HTML Structure
```html
<div class="legalmint-embed" data-theme="light">
  <div class="legalmint-header">
    <h1>Privacy Policy</h1>
    <p class="legalmint-last-updated">Last updated: May 13, 2026</p>
    <p class="legalmint-disclaimer">
      This document was generated by LegalMint AI and is provided for informational purposes only.
    </p>
  </div>
  <div class="legalmint-content">
    <!-- Rendered document content -->
    <h2>1. Introduction</h2>
    <p>YourCompany, Inc. ("we", "our", or "us") is committed to protecting...</p>
    <!-- ... -->
  </div>
  <div class="legalmint-footer">
    <a href="https://legalmint.ai/powered-by" target="_blank" rel="noopener">
      Powered by LegalMint AI
    </a>
  </div>
</div>
```

### JavaScript SDK (`legalmint-embed.js`)

#### Initialization
```javascript
(function() {
  const containers = document.querySelectorAll('[data-document-id]');
  
  containers.forEach(container => {
    const documentId = container.getAttribute('data-document-id');
    const theme = container.getAttribute('data-theme') || 'light';
    const language = container.getAttribute('data-language') || 'en';
    
    // Attach Shadow DOM
    const shadow = container.attachShadow({ mode: 'open' });
    
    // Load document from cache or API
    loadDocument(documentId, language)
      .then(document => renderDocument(shadow, document, theme))
      .catch(err => console.warn('LegalMint embed failed:', err));
  });
})();
```

#### Auto-Update Mechanism
- On load, checks localStorage for a cached document + timestamp
- If cached version is < 6 hours old, serves from cache
- Performs a background HEAD request to `/api/embed/{documentId}/version` to check if a newer version exists
- If newer version exists, fetches the full document and updates the DOM seamlessly (no flicker)
- Uses `requestIdleCallback()` or `setTimeout()` to avoid blocking the host page's main thread

#### Security
- All API requests include the `data-client-id` as an auth token in the header
- API rate-limited to 10 requests per hour per client ID
- Embed only serves documents explicitly marked as "published" — never drafts
- CORS headers applied on the API endpoint: `Access-Control-Allow-Origin: *` (public embed API)

## 7. Edge Cases

### Very Long Documents
- **PDF:** Puppeteer handles any length natively via pagination. TOC page count is estimated and adjusted. If document exceeds 200 pages, the generation time is extended to allow up to 90 seconds (vs. default 30).
- **DOCX:** OOXML format has no practical page limit. Word handles documents of any length.
- **Embed:** Documents > 50KB are chunked; the embed loads visible sections first and lazy-loads the rest on scroll.

### Special Characters and Unicode
- All character handling uses UTF-8 throughout the pipeline
- **PDF:** Puppeteer/Chromium natively handles Unicode. Fonts must include the required character ranges. Fallback: Noto Sans as a comprehensive Unicode font.
- **DOCX:** OOXML supports Unicode fully. Asian character sets, Arabic, Hebrew, and emoji are all supported.
- **Embed:** HTML page encoding is `UTF-8`. The embed script includes `charset="utf-8"` in its API requests.
- **Test characters required:** ™ © ® — " ' € £ ¥ — em-dash – en-dash é ñ ü 中文 العربية

### Embedded Links in PDF
- All hyperlinks in the source Markdown become clickable links in the PDF (Puppeteer preserves `<a href>` elements)
- Internal TOC links navigate to the corresponding page within the PDF
- External links open in the user's default browser (PDF reader behavior)
- Link text is styled distinctly (blue, underlined) in the PDF

### Large Files and Performance
- PDF generation is queued for documents expected to exceed 5MB; user sees "Generating your PDF... This may take up to 90 seconds."
- DOCX generation is synchronous and fast (< 5 seconds for standard documents)
- Generated files are cached with a TTL of 24 hours or until the document is updated
- If the same user requests the same document export within the cache window, the cached file is served immediately

### Concurrent Export Requests
- Multiple export requests for the same document from the same user are coalesced (only one generation job runs)
- Rate limiting: 10 exports per hour per user across all formats

### Document With Missing Sections
- If a document has sections the user removed (or never generated), the export includes only the existing sections
- A note is added at the bottom: "This document does not include all recommended sections. Review your document in LegalMint AI."

### Multi-Language Exports
- Templates are localized; all exported text is in the selected language
- PDF fonts must support the target language's character set
- TOC and page numbering use the selected language's numbering conventions
- Right-to-left (RTL) languages (Arabic, Hebrew) trigger RTL page layout in PDF and DOCX

## 9. API Endpoints

### Export API

| Method | Endpoint | Description |
|--------|----------|-------------|
| `POST` | `/api/export/{documentId}/pdf` | Generate PDF (queued, returns job ID) |

| `POST` | `/api/export/{documentId}/docx` | Generate and download DOCX (sync) |
